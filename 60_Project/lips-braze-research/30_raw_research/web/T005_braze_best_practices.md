## 2025年リテール顧客エンゲージメントレビュー

### 主要トレンド

1. **フィジタル（Phygital）体験**
   - オンライン/オフラインを統合したエンゲージメント
   - 顧客は「販売チャネル間のシームレスなコミュニケーション」を期待
   - 「ジャーニーが記録され、データが追従する」体験

2. **データドリブンパーソナライゼーション**
   - リアルタイム顧客メッセージのパーソナライズ
   - より効率的なデータ分析戦略

3. **組織的エクセレンス**
   - 内部実験とクロスファンクショナル協業のベストプラクティス

### 統計データ

| 項目 | 数値 |
|------|------|
| 感情的つながりを重視し収益目標超過したブランド | 60% |
| 3チャネル以上を使用するトップパフォーマー | 16%多い |
| 統合プラットフォームを使用するブランド | 50% |
| データ共有の同意に懸念を持つマーケター | 43% |
| AIでデータ分析を行うブランド | 39% |
| AIで顧客感情分析を行うブランド | 38% |
| 感情的インパクトに不安を持つマーケター | 85% |
| 感情的共鳴のためにテクノロジーを使用するリテーラー | 91% |

### 推奨アプローチ
- 教育から始まり、ソーシャルプルーフを追加、コンバージョンプロンプトを導入する段階的ワークフロー
- メール、in-app、広告の調整
- コンテンツを購買ステージにマッピング
- 疲労を避けるための送信間隔設定

## Brazeキャンバス ベストプラクティス

### 基本フレームワーク: 3つの問い
1. **What**: ユーザーに何を達成してもらいたいか
2. **Who**: リーチしようとしているユーザーは誰か
3. **Why**: なぜこのキャンバスが必要か

### 設計戦略

#### コンポーネントの活用
- **条件分岐**: ユーザーを異なるワークフローにセグメント
- **遅延ステップ**: ユーザージャーニーの間隔設定
- **スタンドアロンメッセージ**: 任意の場所にメッセージ追加
- **Content Cards**: プロモーションオファー、プッシュ通知用コンテンツ

#### Canvas Flow の特徴
- 直感的UIで迅速なジャーニー設計
- アクションパス、オーディエンスパス、条件分岐
- 状況に即した嗜好・行動・クロスチャネルインタラクションに基づく自動調整
- リアルタイムデータ取得と予測分析

### テスト・最適化

1. **コントロールグループ**: キャンバスの効果検証
2. **実験パス**: ランダム割り当てによるABテスト
3. **明確な命名**: ステップ・バリアントの識別
4. **キャンバス分析**: コンポーネント別パフォーマンス確認

### 実装上の注意
- スケジュール変更は「まだキューに入っていないユーザー」にのみ適用
- メッセージスキップや配信失敗を防ぐ設計が必要

### ユーザーエントリー方法
1. **スケジュール**: 時間ベース
2. **アクションベース**: 定義されたアクション実行後
3. **APIトリガー**: APIリクエスト経由

## プッシュ通知 ベストプラクティス

### プラットフォーム別考慮事項

#### iOS
- 明示的オプトインが必要（OS レベルのシステムプロンプト）
- ユーザーの決定は最終的（拒否後の再要求不可）
- **プッシュプライマーIAM**がオプトイン率向上の重要戦略
- **Provisional Authorization**: ネイティブプロンプト前の「お試しパス」

#### Android
- Android 13以降: 明示的許可が必要
- Android 12以下: 最初のセッションで自動購読
- アプリが通知チャネルを作成時に自動プロンプト

### プッシュプライマー戦略

#### タイミング
> 「理想的なタイミングは、ユーザーが価値あるアクションを完了した時。アプリやサイトで価値を感じ始めた時、またはプッシュ通知で対応できる説得力のあるニーズがある時」

- 例: 注文後に配送追跡情報を提供したい時
- 顧客にとって有益なタイミングで要求

#### コンテンツ
- プッシュ通知を有効にすべき理由を明確に説明
- 送信する通知の種類と提供する価値を具体的に

#### ターゲティング
- フィルター: 「Foreground Push Enabled For App is false」
- 例: 2回目の購入完了ユーザー、会員登録直後のユーザー、週2回以上訪問するユーザー

### iOS Provisional Authorization
- オプトイン率を劇的に向上させる可能性
- 「Keep」または「Turn Off」の選択肢
- 価値あるメッセージの提供が重要

### メッセージガイドライン
- タイトル・本文: 各30-40文字程度を推奨
- Android: 通知は手動削除まで保持
- iOS: 1つの通知を開くと他はロック画面からクリア

### プラットフォーム差異
- Androidユーザーはプッシュ通知へのエンゲージメントが高い傾向
- iOS/Android間でエンゲージメント・リテンション率が異なる
- クロスプラットフォーム分析時に考慮が必要

### アプリ内プッシュ設定
- デバイスレベルでの通知オフを防ぐ
- アプリ内で購読管理を提供
- Brazeのboolean カスタム属性で設定ステータス管理

## LIPSへの示唆

### 優先実装事項
1. **プッシュプライマー戦略**: iOS オプトイン率向上が最重要
2. **Canvas設計**: 「What/Who/Why」フレームワークの適用
3. **段階的ワークフロー**: 教育→ソーシャルプルーフ→コンバージョン
4. **コントロールグループ**: 全施策で効果検証を必須化
5. **フィジタル体験**: マツキヨとの連携でオフライン統合

### KPI設定の参考
- 3チャネル以上の活用でトップパフォーマー入り
- 感情的つながりの重視が収益目標超過に直結
- AI活用によるデータ分析・感情分析の導入
