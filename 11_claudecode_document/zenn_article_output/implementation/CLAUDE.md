# プロジェクト設定

このファイルは Claude Code がプロジェクトを理解するための設定ファイルです。
プロジェクトに合わせてカスタマイズしてください。

## 開発環境

- **言語**: TypeScript
- **フレームワーク**: Next.js / Express / etc.
- **ランタイム**: Node.js v20.x
- **パッケージマネージャー**: npm / pnpm / yarn

## コーディング規約

### 命名規則
| 対象 | 規則 | 例 |
|------|------|-----|
| 変数・関数 | camelCase | `getUserName`, `isValid` |
| クラス・型 | PascalCase | `UserService`, `ApiResponse` |
| 定数 | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT` |
| ファイル | kebab-case | `user-service.ts` |

### コメント言語
- **コードコメント**: 日本語
- **コミットメッセージ**: 日本語
- **PR説明**: 日本語

### TypeScript ルール
- `any` の使用を避ける（やむを得ない場合は理由をコメント）
- 明示的な型定義を推奨
- `strict` モードを有効化

## テスト

- **テストフレームワーク**: Jest / Vitest
- **カバレッジ目標**: 80%以上
- **テストファイル命名**: `*.test.ts` / `*.spec.ts`
- **テストの配置**: `__tests__/` ディレクトリ または 対象ファイルと同階層

## セキュリティ要件

- 環境変数での機密情報管理（`.env` は `.gitignore` に追加）
- SQLインジェクション対策（プリペアドステートメント使用）
- XSS対策（ユーザー入力のエスケープ処理）
- CSRF対策（トークン検証）
- 認証情報のハードコーディング禁止

## PRレビュー基準

PRがマージ可能となる条件：

1. **テスト**
   - すべてのテストが通過すること
   - 新機能にはテストが含まれること

2. **品質**
   - 型エラーがないこと
   - リンターエラーがないこと
   - コードフォーマットが適用されていること

3. **ドキュメント**
   - 公開APIには適切なJSDocコメントがあること
   - 破壊的変更がある場合は README を更新すること

## ディレクトリ構造

```
src/
├── components/     # UIコンポーネント
├── hooks/          # カスタムフック
├── services/       # ビジネスロジック
├── utils/          # ユーティリティ関数
├── types/          # 型定義
└── __tests__/      # テストファイル
```
