# Skills ディレクトリ構成

## 概要

Skillsはサポートリソースをバンドルすることで強力になる。標準的な構造は3つのディレクトリを使用し、それぞれが特定の目的を果たす。

## 基本構造

```
skill-name/
├── SKILL.md          # 必須: コアプロンプトと指示
├── scripts/          # オプション: 実行可能なPython/Bashスクリプト
├── references/       # オプション: コンテキストにロードするドキュメント
└── assets/           # オプション: テンプレートとバイナリファイル
```

## 各ディレクトリの役割

### SKILL.md（必須）

Skillのエントリーポイント。YAMLフロントマター + Markdown指示で構成。

```yaml
---
name: pdf-processing
description: Extract text and tables from PDF files...
---

# PDF Processing

## Quick Start
[最小限の動作例]

## Operations
[主要な操作]
```

### scripts/（オプション）

**目的**: 実行可能コードをバンドル（ゼロコンテキスト実行）

スクリプトは実行されるだけで、その内容はコンテキストにロードされない。出力のみがトークンを消費する。

```
scripts/
├── analyze_form.py    # フォームフィールド抽出
├── fill_form.py       # フォーム入力
├── validate.py        # バリデーション
└── rotate_pdf.py      # PDF回転
```

**スクリプトの参照方法:**
```markdown
## Utility scripts

**analyze_form.py**: Extract all form fields from PDF

```bash
python scripts/analyze_form.py input.pdf > fields.json
```
```

**重要な区別:**
- **実行する場合**（一般的）: "Run `analyze_form.py` to extract fields"
- **参照として読む場合**: "See `analyze_form.py` for the extraction algorithm"

### references/（オプション）

**目的**: 必要に応じてコンテキストにロードするドキュメント

```
references/
├── REFERENCE.md      # 詳細な技術リファレンス
├── FORMS.md          # フォームテンプレート
├── finance.md        # ドメイン固有（財務）
├── sales.md          # ドメイン固有（営業）
└── api-docs.md       # API仕様
```

**参照ガイドライン:**
- 個々のファイルは100行以下を目安
- 100行超の場合は目次を含める
- SKILL.mdから1レベル深さで参照

### assets/（オプション）

**目的**: 出力に使用する静的リソース（コンテキストにロードされない）

```
assets/
├── templates/
│   ├── report.docx
│   └── invoice.xlsx
├── logos/
│   └── company-logo.png
├── hello-world/
│   ├── index.html
│   └── app.jsx
└── schemas/
    └── data-schema.json
```

## 実践的な構成例

### 例1: PDFスキル

```
pdf/
├── SKILL.md              # メイン指示
├── FORMS.md              # フォーム入力ガイド
├── reference.md          # APIリファレンス
├── examples.md           # 使用例
└── scripts/
    ├── analyze_form.py   # フィールド抽出
    ├── fill_form.py      # フォーム入力
    └── validate.py       # バリデーション
```

### 例2: BigQueryスキル

```
bigquery-skill/
├── SKILL.md              # 概要とナビゲーション
└── references/
    ├── finance.md        # 収益、ARR、請求
    ├── sales.md          # 機会、パイプライン
    ├── product.md        # API使用状況
    └── marketing.md      # キャンペーン、アトリビューション
```

### 例3: ドキュメント生成スキル

```
document-generator/
├── SKILL.md
├── references/
│   ├── style-guide.md
│   └── formatting.md
├── assets/
│   ├── templates/
│   │   ├── proposal.docx
│   │   └── report.docx
│   └── logos/
│       └── company.png
└── scripts/
    └── validate_doc.py
```

## 含めるべきでないもの

以下の補助的ドキュメントは**作成しない**：

- README.md
- INSTALLATION_GUIDE.md
- QUICK_REFERENCE.md
- CHANGELOG.md

Skillには「AIエージェントがタスクを実行するために必要な情報」のみを含める。

## ファイル参照のベストプラクティス

### 相対パスを使用

```markdown
# 良い例
See [the reference guide](references/REFERENCE.md) for details.
Run: `python scripts/extract.py`

# 悪い例（絶対パス）
See [the reference guide](/home/user/skills/pdf/references/REFERENCE.md)
```

### Unixスタイルパス（フォワードスラッシュ）

```markdown
# 良い例
scripts/helper.py
references/guide.md

# 悪い例（Windowsスタイル）
scripts\helper.py
references\guide.md
```

### 1レベル深さを維持

```markdown
# 良い例
SKILL.md → advanced.md
SKILL.md → reference.md
SKILL.md → examples.md

# 悪い例
SKILL.md → advanced.md → details.md → actual_info.md
```

## ファイル命名規則

**推奨:**
- 内容を示す記述的な名前: `form_validation_rules.md`
- 小文字、ハイフン区切り: `api-reference.md`

**避ける:**
- 曖昧な名前: `doc2.md`, `file1.md`
- 大文字混在: `FormValidation.md`
