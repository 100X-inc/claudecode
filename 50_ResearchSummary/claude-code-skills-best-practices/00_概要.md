# Claude Code Skills ベストプラクティス - 調査概要

## エグゼクティブサマリー

Agent Skillsは、Claudeに専門知識・ワークフロー・ツール連携を教えるための「動的にロードされる指示書フォルダ」である。Anthropicは2025年10月にこの機能を発表し、Claude Code、Claude.ai、APIで利用可能となった。

## 主要な発見

### Skills とは

- **定義**: 指示、スクリプト、リソースを含むフォルダで、Claudeが特定タスクのパフォーマンスを向上させるために動的にロードする
- **用途**: 企業のブランドガイドラインに沿ったドキュメント作成、組織固有のワークフローによるデータ分析、個人タスクの自動化など
- **必須ファイル**: `SKILL.md` のみ（YAMLフロントマター + Markdown本文）

### 核心的設計原則: Progressive Disclosure（段階的開示）

Skillsの最も重要な設計原則。マニュアルの目次→章→付録のように、必要な情報だけを段階的にロードする：

1. **Tier 1**: メタデータ（name/description）がシステムプロンプトにプリロード（約100トークン）
2. **Tier 2**: Skill発動時にSKILL.md本文をロード（5000トークン以下推奨）
3. **Tier 3+**: 必要に応じて参照ファイル（references/, scripts/等）をロード

### SKILL.md の構造

```yaml
---
name: skill-name           # 必須: 最大64文字、小文字・数字・ハイフンのみ
description: 説明文        # 必須: 最大1024文字、何をするか＋いつ使うか
---

# Skill名

[Claudeが従う指示をここに記述]

## Examples
- 使用例1
- 使用例2
```

### ベストプラクティスの要点

| カテゴリ | 推奨事項 |
|---------|---------|
| 簡潔さ | SKILL.md本文は500行以下、超えたら分割 |
| 命名 | 動名詞形（processing-pdfs）を推奨 |
| 説明文 | 三人称で記述、トリガーキーワードを含める |
| 自由度 | タスクの脆弱性に応じて指示の詳細度を調整 |
| テスト | Haiku/Sonnet/Opus全モデルでテスト |

## 結論

Skillsは「Claudeへのオンボーディングガイド」として機能し、専門知識・手順的ワークフロー・再利用可能なツールを装備させる。Progressive Disclosureにより、事実上無制限のコンテキストをバンドル可能でありながら、トークン消費を最小限に抑える。
