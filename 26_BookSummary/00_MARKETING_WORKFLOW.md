## 1. クイックスタートガイド

### 30秒診断チェックリスト

以下の5つの質問に答えて、状況コードを特定する:

1. **ビジネスタイプ**: B2B Enterprise / B2B SMB / B2C High-Involvement / B2C Low-Involvement
2. **会社ステージ**: Pre-PMF / Early Growth / Scaling / Maturity / Enterprise
3. **市場ポジション**: Leader / Challenger / Nicher / Follower
4. **製品ライフサイクル**: Innovation / Chasm / Growth / Maturity / Decline
5. **主要課題**: Awareness / Acquisition / Retention / Expansion / Positioning

**状況コード形式**: `{BusinessType}-{Stage}-{Position}-{Lifecycle}-{Challenge}`

例: `B2B-SMB-Scaling-Challenger-Growth-Acquisition`

### 即座に使えるフレームワーク推奨

```
質問: 今最も緊急の課題は何か?

├─ 認知が足りない → PAD (Preference, Awareness, Distribution)
├─ リードが足りない → THE MODEL + デマンドジェネレーション
├─ 受注率が低い → StoryBrand + 5A最適化
├─ 解約が多い → Jobs to be Done + カスタマーサクセス
└─ 差別化できない → 4S Check + ポジショニング再設計
```

## 2. 状況分類システム（5次元）

### 2.1 ビジネスタイプ（Business Type）

| 分類 | 定義 | 特徴 | 代表的指標 |
|-----|------|------|-----------|
| **B2B Enterprise** | 大企業向けビジネス | 長い販売サイクル、複数意思決定者、高単価 | ACV $50K+, 販売サイクル6-12ヶ月 |
| **B2B SMB** | 中小企業向けビジネス | 短い販売サイクル、単独意思決定、中単価 | ACV $1K-50K, 販売サイクル1-3ヶ月 |
| **B2C High-Involvement** | 高関与消費財 | 慎重な検討、情報収集、感情的要素大 | 車、住宅、教育、保険 |
| **B2C Low-Involvement** | 低関与消費財 | 習慣的購買、価格感度高、ブランド惰性 | 日用品、食品、飲料 |

**診断質問**:
- 顧客は企業か個人か?
- 意思決定者は1人か複数か?
- 購買前の検討期間は?
- 1回の購買単価は?

### 2.2 会社ステージ（Company Stage）

| 分類 | 定義 | マーケティング焦点 |
|-----|------|-----------------|
| **Pre-PMF** | プロダクト・マーケット・フィット前 | 顧客発見、仮説検証、ジョブ理解 |
| **Early Growth** | 初期成長期（PMF直後） | 再現可能なチャネル発見、単価最適化 |
| **Scaling** | スケーリング期 | 組織構築、プロセス標準化、MOps導入 |
| **Maturity** | 成熟期 | 効率最適化、ブランド強化、新市場探索 |
| **Enterprise** | 大企業期 | ガバナンス、多事業統合、M&A |

**診断質問**:
- 顧客からの自然な紹介・口コミがあるか?（PMF指標）
- 同じ方法で繰り返し顧客獲得できているか?
- マーケティングプロセスは標準化されているか?
- 複数事業・複数地域を運営しているか?

### 2.3 市場ポジション（Market Position）

| 分類 | 定義 | 推奨戦略 |
|-----|------|---------|
| **Leader** | 市場シェア1位 | シェア防衛、市場拡大、価格決定力 |
| **Challenger** | 市場シェア2-3位 | リーダー攻撃、差別化、ニッチ奪取 |
| **Nicher** | 特定セグメント特化 | セグメント深耕、専門性強化、プレミアム化 |
| **Follower** | 市場追随者 | コスト効率、模倣、ニッチ探索 |

**診断質問**:
- 市場シェアは何位か?
- 競合と比較した認知度は?
- 価格決定力はあるか?
- 特定セグメントで強いか?

### 2.4 製品ライフサイクル（Product Lifecycle）

| 分類 | 定義 | 顧客セグメント | 推奨アプローチ |
|-----|------|--------------|--------------|
| **Innovation** | 革新期 | イノベーター（2.5%） | テクノロジー訴求、ビジョン共有 |
| **Chasm** | キャズム期 | アーリーアドプター→アーリーマジョリティ | D-Day戦略、ホールプロダクト |
| **Growth** | 成長期 | アーリーマジョリティ（34%） | 市場拡大、チャネル最適化 |
| **Maturity** | 成熟期 | レイトマジョリティ（34%） | 差別化、コスト効率、ブランド |
| **Decline** | 衰退期 | ラガード（16%） | 収穫、撤退計画、次世代準備 |

**診断質問**:
- 製品カテゴリ自体の成長率は?
- 顧客は「なぜこれが必要か」を理解しているか?
- 競合は何社あるか?
- 導入事例を求められるか?

### 2.5 主要課題（Primary Challenge）

| 分類 | 定義 | 5Aステージ | 主要KPI |
|-----|------|----------|--------|
| **Awareness** | 認知課題 | Aware | ブランド認知率、Aided/Unaided想起 |
| **Acquisition** | 獲得課題 | Appeal → Ask → Act | MQL数、SAL率、受注率 |
| **Retention** | 維持課題 | Act → Advocate | 更新率、NPS、チャーン率 |
| **Expansion** | 拡大課題 | Advocate | アップセル率、LTV、紹介率 |

**診断質問**:
- 見込み客は自社を知っているか?

- 競合との違いを説明できるか?

## 3. 状況→ニーズマッピング

### 3.1 Challenge別の課題・成果・リソース表

#### Awareness（認知課題）

| 項目 | 内容 |
|-----|------|
| **最も切実な問題** | ターゲット顧客が自社・製品を知らない |
| **必要な成果** | 認知率向上、エボークトセット入り |
| **主要フレームワーク** | PAD（Preference, Awareness, Distribution） |
| **補完フレームワーク** | コンテンツマーケティング、5A認知ステージ |
| **必要リソース** | マーケティング予算、クリエイティブ、PR |
| **成功指標** | Aided認知率、第1想起率、検索ボリューム |

**PADの適用**:
```
売上 = プレファレンス × 認知 × 配荷

認知課題の場合:
- 認知率 50% → 80% で売上1.6倍
- Aided認知を優先（均一性が高く測定容易）
- 第1想起率はエボークトセットとの相関が高い
```

#### Acquisition（獲得課題）

| 項目 | 内容 |
|-----|------|
| **最も切実な問題** | リードはあるが受注に至らない / リードが足りない |
| **必要な成果** | MQL→SAL→SQL→受注の転換率向上 |
| **主要フレームワーク** | THE MODEL（B2B）/ 5Aモデル（B2C） |
| **補完フレームワーク** | StoryBrand、ABM、キャズム |
| **必要リソース** | インサイドセールス、MA/CRM、コンテンツ |
| **成功指標** | SAL率、受注率、CAC、販売サイクル |

**THE MODELの適用（B2B）**:
```
デマンドウォーターフォール:
リード → MQL → SAL → SQL → パイプライン → 受注

ボトルネック診断:
- MQL→SAL転換率 < 50%: 営業との溝（ABM導入）
- SAL→SQL転換率 < 30%: リード品質（スコアリング改善）
- SQL→受注率 < 20%: 商談力（フェーズ3強化）
```

**5Aモデルの適用（B2C）**:
```
5A: Aware → Appeal → Ask → Act → Advocate

Acquisition課題の焦点:
- Appeal→Ask: 興味喚起（コンテンツ）
- Ask→Act: 確信形成（Fファクター活用）
- PAR = Act ÷ Appeal: 購買行動率
```

#### Retention（維持課題）

| 項目 | 内容 |
|-----|------|
| **最も切実な問題** | 顧客が離脱する、更新しない |
| **必要な成果** | 更新率向上、チャーン率低減 |
| **主要フレームワーク** | カスタマーサクセス、Jobs to be Done |
| **補完フレームワーク** | THE MODEL（CS部分）、LTV設計 |
| **必要リソース** | CSチーム、ヘルススコア、オンボーディング |
| **成功指標** | 更新率、NPS、ヘルススコア、活用度 |

**Jobs to be Doneの適用**:
```
顧客が「雇用」する理由を理解:

質問: 顧客はどんなジョブを片づけるために製品を雇用しているか?

3つの側面:
- 機能面: 実用的なニーズ
- 社会面: 他者からどう見られたいか
- 感情面: どう感じたいか

→ ジョブが解決されていないなら離脱する
```

#### Expansion（拡大課題）

| 項目 | 内容 |
|-----|------|
| **最も切実な問題** | 既存顧客からの追加売上が少ない |
| **必要な成果** | クロスセル・アップセル・紹介の増加 |
| **主要フレームワーク** | ABM（面でつながる）、LTV最大化 |
| **補完フレームワーク** | 5A Advocate、カスタマーサクセス |
| **必要リソース** | アカウントプラン、CS/営業連携 |
| **成功指標** | LTV、アップセル率、紹介率、BAR |

**ABM「面でつながる」戦略**:
```
関係構築の進化:
- 点: 1営業→1部署→1製品
- 線: 部署としての関係
- 面: 複数営業→多部署→多製品

ABMでターゲットアカウントのLTV最大化:
1. 既存顧客の全部署をマッピング
2. 未取引部署にクロスセル機会を特定
3. 複数製品・サービスの組み合わせ提案
```

#### Positioning（差別化課題）

| 項目 | 内容 |
|-----|------|
| **最も切実な問題** | 競合との違いを説明できない |
| **必要な成果** | 明確な差別化、価格プレミアム |
| **主要フレームワーク** | StoryBrand、4Sチェック |
| **補完フレームワーク** | キャズム（ポジショニング）、Consumer Driven |
| **必要リソース** | 市場調査、メッセージ開発、ブランディング |
| **成功指標** | ブランド選好度、価格プレミアム、Win率 |

**StoryBrandの適用**:
```
7つの要素でメッセージ再構築:

1. 主人公（顧客）: 顧客が望むものを定義
2. 問題の特定: 外的・内的・哲学的問題
3. 導き手の登場: 共感＋権威
4. 計画の提示: プロセス計画＋約束計画
5. 行動喚起: 直接的CTA＋移行的CTA
6. 回避したい失敗: 何を失うか
7. 成功する結末: 勝利・統合・最高の自分
```

### 3.2 フレームワーク推奨マトリクス

| ビジネスタイプ | Awareness | Acquisition | Retention | Expansion | Positioning |
|--------------|-----------|-------------|-----------|-----------|-------------|
| **B2B Enterprise** | コンテンツMkt + ABM | THE MODEL + ABM | CS + JtBD | ABM面戦略 | StoryBrand + 4S |
| **B2B SMB** | PAD + デジタル | THE MODEL | CS + JtBD | LTV設計 | StoryBrand |
| **B2C High-Inv** | 5A + PR | 5A + StoryBrand | JtBD + CRM | 5A Advocate | StoryBrand + 4S |
| **B2C Low-Inv** | PAD + 配荷 | プレファレンス | NBDモデル | 水平拡大 | プレファレンス |

## 4. 理論統合システム

### 4.1 フレームワーク選択デシジョンツリー

```
START: 主要課題は何か?
│
├─ Awareness（認知）
│   ├─ B2B → コンテンツMkt + ABM + THE MODEL(マーケ部分)
│   └─ B2C → PAD + 5A(Aware) + PR戦略
│
├─ Acquisition（獲得）
│   ├─ B2B Enterprise
│   │   ├─ Chasm期 → D-Day戦略 + ホールプロダクト
│   │   └─ Growth期 → THE MODEL + ABM
│   ├─ B2B SMB → THE MODEL + インサイドセールス
│   ├─ B2C High-Inv → 5Aモデル + StoryBrand
│   └─ B2C Low-Inv → プレファレンス + 配荷戦略
│
├─ Retention（維持）
│   ├─ B2B → カスタマーサクセス + JtBD + ヘルススコア
│   └─ B2C → JtBD + CRM + ロイヤルティプログラム
│
├─ Expansion（拡大）
│   ├─ B2B → ABM(面戦略) + クロスセル設計
│   └─ B2C → 5A Advocate + 水平拡大 + 紹介プログラム
│
└─ Positioning（差別化）
    ├─ メッセージ課題 → StoryBrand 7要素
    ├─ 戦略課題 → 4Sチェック + Consumer Driven
    └─ 市場課題 → キャズム・ポジショニング
```

### 4.2 4層フレームワークスタック

マーケティング活動を4層で体系的に設計する。

```
┌─────────────────────────────────────────────────────────┐
│ Layer 4: 計測層（Measurement）                          │
│ ─────────────────────────────────────────────────────── │
│ PAR/BAR | MQL-SAL-SQL | 90日サイクル | SLA | CMMIモデル │
├─────────────────────────────────────────────────────────┤
│ Layer 3: 戦術層（Tactical Execution）                   │
│ ─────────────────────────────────────────────────────── │
│ StoryBrand | 5Aモデル | コンテンツMkt | MOps | ABM実行  │
├─────────────────────────────────────────────────────────┤
│ Layer 2: 戦略層（Market Strategy）                      │
│ ─────────────────────────────────────────────────────── │
│ キャズム/D-Day | THE MODEL | PAD | ABM戦略 | LTV設計    │
├─────────────────────────────────────────────────────────┤
│ Layer 1: 哲学層（Strategic Foundation）                 │
│ ─────────────────────────────────────────────────────── │
│ Consumer Driven | Jobs to be Done | コトラー | 4Sチェック│
└─────────────────────────────────────────────────────────┘
```

### 4.3 フレームワーク組み合わせルール

#### 推奨ペアリング

| 状況 | 主フレームワーク | 必須補完 | 理由 |
|-----|----------------|---------|------|
| B2B Enterprise + Acquisition | THE MODEL | ABM | 営業視点のターゲット定義でSAL率向上 |
| B2B + Chasm期 | D-Day戦略 | StoryBrand | 橋頭堡攻略にはメッセージの明確化が必須 |
| B2C High-Involvement | 5Aモデル | StoryBrand | Ask段階での物語が購買決定を促進 |
| B2C Low-Involvement | プレファレンス/PAD | NBDモデル | 習慣的購買の確率予測が有効 |
| Any + Retention | カスタマーサクセス | Jobs to be Done | 離脱の根本原因はジョブ未解決 |
| Any + Positioning | StoryBrand | 4Sチェック | メッセージ×戦略の両輪で差別化 |

#### アンチパターン（避けるべき組み合わせ）

| アンチパターン | 問題 | 代替案 |
|--------------|------|-------|
| ABM without THE MODEL | 営業プロセスなしにABMは機能しない | 先にTHE MODEL基盤構築 |
| D-Day without ホールプロダクト | 橋頭堡で完全解決策なしは失敗する | パートナー連携でホールプロダクト構築 |
| PAD without Consumer Driven | 数字最適化が顧客不在になる | 4Sチェックで顧客価値確認 |
| THE MODEL without SLA | プロセスあってもボトルネック不明 | MOps導入でSLA設定 |
| StoryBrand without 調査 | 推測ベースのメッセージは響かない | ジョブ調査で顧客理解 |

### 4.4 スタック構築の手順

```
Step 1: 哲学層の選択
├─ Consumer Driven: 自社は本当に消費者視点か?
├─ 4Sチェック: 戦略はSelective, Sufficient, Sustainable, Synchronizedか?
└─ Jobs to be Done: 顧客のジョブを理解しているか?

Step 2: 戦略層の選択
├─ 状況診断コードを参照
├─ ビジネスタイプに適した戦略フレームワークを選択
└─ 複数該当する場合は主要課題で優先順位付け

Step 3: 戦術層の選択
├─ 戦略層で選んだフレームワークに対応する戦術を選択
├─ StoryBrandでメッセージを統一
```

### Phase 2: 戦略選択フェーズ（Strategy Selection Phase）

**期間目安**: 1-2週間

**目的**: 最適なフレームワークを選択し、戦略の妥当性を検証する

#### Step 2.1: フレームワーク選択

- [ ] デシジョンツリーを使用して主フレームワークを選択

#### Step 2.2: 4Sチェックの実施

**Selective（選択的か?）**:
- [ ] やることとやらないことが明確に区別されているか?

- [ ] 「全部やる」になっていないか?

**Sufficient（十分か?）**:
- [ ] 投入する経営資源が勝利に十分か?
- [ ] 「勝てるライン」に到達できる投資か?
- [ ] 中途半端な分散になっていないか?

**Sustainable（継続可能か?）**:
- [ ] 中長期で維持継続できるか?
- [ ] 競合が真似できない要素はあるか?
- [ ] 自社資源が枯渇しないか?

**Synchronized（自社特徴との整合性）**:
- [ ] 自社の強みを活かしているか?
- [ ] 自社の弱みに依存していないか?
- [ ] 自社の強みが競合の弱みにハマるか?

## 7. フレームワーク早見カード

### 7.1 PAD（Preference, Awareness, Distribution）

**出典**: [[確率思考の戦略論]]

**核心**: ビジネスを伸ばすための経営資源配分先は3つに集約される

```
売上 = プレファレンス × 認知 × 配荷

┌─────────────────────────────────────────────┐
│ Preference（プレファレンス/好意度）            │
│ ・ブランドの最大ポテンシャルを決定             │
│ ・無限の可能性を持つ                         │
│ ・構成要素: ブランド・エクイティ、価格、製品性能│
├─────────────────────────────────────────────┤
│ Awareness（認知）                            │
│ ・0〜100%の世界                             │
│ ・認知50%→80%で売上1.6倍                    │
│ ・Aided認知が需要予測に最重要                 │
├─────────────────────────────────────────────┤
│ Distribution（配荷）                          │
│ ・0〜100%の世界                             │
│ ・質の改善: SKU数、棚位置、価格設定           │
└─────────────────────────────────────────────┘
```

**適用場面**: 認知課題、B2C Low-Involvement、リソース配分決定

**成功事例**: USJハリー・ポッター（認知90%以上で200万人追加集客）

---

### 7.2 THE MODEL

**出典**: [[THE-MODEL-マーケティング・インサイドセールス・営業・カスタマーサクセスの共業プロセス]]

**核心**: マーケティング・インサイドセールス・営業・カスタマーサクセスの分業から共業へ

```
┌──────────────┐   ┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│ マーケティング │ → │インサイドセールス│ → │  営業(FS)   │ → │カスタマーサクセス│
├──────────────┤   ├──────────────┤   ├──────────────┤   ├──────────────┤
│ リード獲得    │   │ リードフォロー  │   │ 商談クロージング│   │ オンボーディング│
│ MQL創出      │   │ 商談化         │   │ 提案・契約    │   │ 活用促進     │
│ ナーチャリング │   │ SQL化         │   │              │   │ 更新・アップセル│
└──────────────┘   └──────────────┘   └──────────────┘   └──────────────┘
        ↓                 ↓                 ↓
      MQL              SAL/SQL          パイプライン         LTV
```

**重要指標**:
- SAL率: 営業が受け入れた案件の割合（目標: 80%+）
- フォーキャスト精度: 四半期±5%、月次±10%
- パイプライン必要額 = 売上目標 ÷ 平均受注率

**適用場面**: B2B全般、特にSaaS

### 7.3 キャズム/D-Day戦略

**出典**: [[キャズム-Ver2]]

**核心**: アーリーアドプターとアーリーマジョリティの間の深い溝を越える

```
テクノロジー・ライフサイクル

 イノベーター  アーリー    アーリー    レイト     ラガード
   2.5%      アドプター   マジョリティ  マジョリティ   16%
             13.5%       34%        34%
    ↓          ↓          ↓          ↓          ↓
 [      ]  [      ]  ⚡[      ]  [      ]  [      ]
                      ↑
                   キャズム
                   （深い溝）
```

**D-Day戦略の4ステップ**:
1. **攻略地点の決定**: 特定ニッチ市場を橋頭堡に
2. **部隊の集結**: ホールプロダクト構築
3. **戦線の見定め**: 競争優位のポジショニング
4. **作戦の実行**: チャネル選択と価格設定

**適用場面**: 新製品のマジョリティ移行、イノベーション製品

### 7.4 StoryBrand 7要素

**出典**: [[ストーリーブランド戦略]]

**核心**: 顧客を主人公に、企業を導き手にしたメッセージング

```
┌─────────────────────────────────────────────────┐
│ 1. 主人公（顧客）が何かを望んでいる              │
│    → 顧客が望むものを1つに定義                  │
├─────────────────────────────────────────────────┤
│ 2. 問題に直面する                              │
│    → 外的・内的・哲学的問題 + 悪役              │
├─────────────────────────────────────────────────┤
│ 3. 導き手（企業）が現れる                       │
│    → 共感 + 権威                               │
├─────────────────────────────────────────────────┤
│ 4. 計画を提示する                              │
│    → プロセス計画 + 約束計画                    │
├─────────────────────────────────────────────────┤
│ 5. 行動を呼びかける                            │
│    → 直接的CTA + 移行的CTA                     │
├─────────────────────────────────────────────────┤
│ 6. 失敗を回避させる                            │
│    → 購入しないと何を失うか                     │
├─────────────────────────────────────────────────┤
│ 7. 成功に導く                                  │
│    → 勝利・統合・最高の自分                     │
└─────────────────────────────────────────────────┘
```

**反応テスト**: 5秒で以下に答えられるか?
1. どんな商品・サービスを提供しているか?
2. それによって買い手の生活はどう改善するか?
3. それを購入するにはどうすればよいか?

**適用場面**: Positioning課題、メッセージ開発、ウェブサイト改善

### 7.5 5Aモデル & PAR/BAR

**出典**: [[コトラーのマーケティング4.0-スマートフォン時代の究極法則]]

**核心**: デジタル時代のカスタマージャーニーと測定指標

```
5A: Aware → Appeal → Ask → Act → Advocate
    認知     訴求    調査   行動    推奨
     ↓        ↓       ↓      ↓       ↓
   知る    興味を持つ 調べる  買う   勧める

           ┌─────────────────┐
      PAR = │  Act ÷ Appeal  │ = 購買行動率
           └─────────────────┘
           ┌─────────────────┐
      BAR = │ Advocate ÷ Act │ = ブランド推奨率
           └─────────────────┘
```

**4つの産業類型**:
- ドアノブ型: 認知→行動スムーズ、推奨低い（日用品）
- 金魚型: 認知→行動長い、推奨高い（高関与製品）
- 漏斗型: 転換率・推奨とも低い（競争激しい市場）
- トランペット型: 転換率・推奨とも高い（理想）

**適用場面**: B2C全般、カスタマージャーニー最適化

### 7.6 ABM（Account Based Marketing）

**出典**: [[究極のBtoBマーケティングABM]]

**核心**: ターゲットアカウントからの売上最大化を目指す戦略的マーケティング

```
従来のマーケティング          ABM
       ↓                     ↓
   広くリード獲得        →  ターゲットアカウント定義
       ↓                     ↓
   スコアリング          →  アカウント単位でスコア
       ↓                     ↓
   MQL→営業へ           →  営業と共同でリスト作成
       ↓                     ↓
   SAL率 50%以下         →  SAL率 90%+
```

**ABMが成果を上げる7つの理由**:
1. 営業とマーケティングが協業できる
2. マーケティング活動を売上に貢献させられる
3. 既存顧客からいち早く成果を出せる
4. 新規顧客の扉をこじ開けられる
5. 低コストでスタートできる
6. 過去の投資を無駄にしない
7. ターゲットアカウントと面でつながれる

**適用場面**: B2B Enterprise、Acquisition/Expansion課題

### 7.7 Jobs to be Done

**出典**: [[ジョブ理論]]

**核心**: 顧客は「ジョブを片づける」ために製品を雇用する

```
顧客が「雇用」する理由を理解:

質問: どんなジョブを片づけたくて、その商品を雇用するのか?

┌─────────────────────────────────────────┐
│ ジョブの3つの側面                        │
├─────────────────────────────────────────┤
│ 1. 機能面: 実用的・実際的なニーズ        │
│ 2. 社会面: 他者からどう見られたいか      │
│ 3. 感情面: どう感じたいか               │
└─────────────────────────────────────────┘

重要: ジョブは顧客の属性（年齢、性別、収入等）からは予測できない
      同じ人でも状況が変われば異なるジョブが発生
```

**ミルクシェイク事例**:
- 朝のミルクシェイクのジョブ: 長い通勤時間を退屈せずに過ごし、昼まで空腹を感じない
- 競合: 他のミルクシェイクではなく、バナナ、ドーナツ、ベーグル

**適用場面**: Retention課題、新製品開発、顧客理解

### 7.8 MOps（Marketing Operations）

**出典**: [[マーケティングオペレーション-MOps-の教科書]]

**核心**: マーケティング活動の管理体制・プロセス構築・データ/システム活用を推進

```
MOpsの4つの役割:

┌─────────────────────────────────────────┐
│ 1. マーケティングツールの管理・運用       │
│    → テクノロジースタック構築            │
├─────────────────────────────────────────┤
│ 2. プロセス策定とベストプラクティス       │
│    → キャンペーンリクエストフォーム       │
├─────────────────────────────────────────┤
│ 3. データマネジメントと分析              │
│    → BI、DMP、分析基盤                  │
├─────────────────────────────────────────┤
│ 4. マーケティングチームの教育            │
│    → オンボーディング、ナレッジベース     │
└─────────────────────────────────────────┘
```

**ガバナンスモデル3タイプ**:
- 集中管理: 1か所で全て管理（小規模・単一事業向け）
- 分散管理: 各地域・事業部が自由に（大規模・多事業向け）
- ハイブリッド: 中央MOps + 個別マーケター（推奨）

**適用場面**: Scaling期以降、マーケティング組織構築

### 7.9 4Sチェック

**出典**: [[USJを劇的に変えた-たった1つの考え方]]

**核心**: 戦略の良し悪しを判断する4つのモノサシ

```
┌─────────────────────────────────────────┐
│ Selective（選択的か?）                   │
│ ・やることとやらないことが明確           │
│ ・経営資源を集中投下できる状態           │
├─────────────────────────────────────────┤
│ Sufficient（十分か?）                    │
│ ・投入資源が勝利に十分                   │
│ ・「勝てるライン」に到達できる           │
├─────────────────────────────────────────┤
│ Sustainable（継続可能か?）               │
│ ・中長期で維持継続できる                 │
│ ・競合が真似できない                     │
├─────────────────────────────────────────┤
│ Synchronized（自社特徴との整合性）        │
│ ・自社の強みを活かしている               │
│ ・自社の弱みに依存していない             │
└─────────────────────────────────────────┘
```

**使い方**:
- 4つ全てにチェックが必要なわけではない
- どこかに突出した強みがあればOK（ホームラン型戦略）
- 平均的にまあまあでは凡庸な戦略

**適用場面**: 戦略評価、意思決定、リソース配分

---

### 7.10 90日サイクル（10Xメソッド）

**出典**: [[10X-10倍の成果を出す仕組み]]

**核心**: 3ヶ月を1サイクルとして目標達成・振り返り・改善を行う

```
90日サイクルの流れ:

Day 1-7: 計画
├─ Impact Filter作成
├─ チーム編成
└─ マイルストーン設定

Day 8-80: 実行
├─ 週次レビュー
├─ 進捗トラッキング
└─ 調整・改善

Day 81-90: 振り返り
├─ GAIN評価（成長確認）
├─ ABCモデル見直し
└─ 次の90日計画
```

**Impact Filterの7項目**:
1. プロジェクト名
2. 目的（何を達成したいか）
3. 重要性（なぜ重要か）
4. 理想的な成果
5. 最良の結果（行動したら）
6. 最悪の結果（行動しなかったら）
7. 成功の基準（SMART）

**適用場面**: 計画策定、チームマネジメント、継続的改善
