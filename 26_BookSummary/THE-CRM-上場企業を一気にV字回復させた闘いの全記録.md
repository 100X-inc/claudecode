## フレームワーク詳細

### 顧客無視戦略(顧客を無視する戦略)

**定義**:
顧客戦略を導入していないということは、顧客を無視した戦略を実行していると認識すべき。

**成功事例: 旅行会社の出発前キャンセル30%問題**

**背景**:
- 海外旅行の出発前キャンセル率30%
- 申込後のアクション: 請求書送付+チケット送付のみ(計2回)

**施策**:
- 「お申込み御礼シナリオ」作成
- 「お帰りなさいシナリオ」作成
- 出発3ヶ月前から顧客の夢を膨らませる情報提供(現地気温、服装、持ち物、観光情報等)

**結果**:
- 出発前キャンセル: 30% → 10%に減少(3分の1に削減)
- 3ヶ月後の海外旅行予約数: 1アップ

**前提条件/注意点**:
企業視点「顧客は大切にしている」 ≠ 顧客視点「売ったらおしまい」

### 顧客理念と行動指針(憲法)

**定義**:
経営者から新入社員まで足並みを揃えて実行する、顧客に対するブレない政策の基盤。

**GDOの顧客理念全文**:
- **方針**: ゴルフの多様な楽しみ方を提案し、ゴルフライフのベストキャディのような存在
- **提供価値**: 単なる情報提供や予約サービスでなく、ゴルフライフをもっと楽しくする豊かな情報・サービス
- **想い**: ゴルフを愛してやまない、ゴルフの楽しさ・楽しみ方を余すことなく伝えたい
- **顧客との関係**: 顧客は私たちにとって顧客であると同時に大切なゴルフ仲間のような存在

**行動基準**:
- 紳士であれ
- 親切であれ
- 有益であれ
- 誠実であれ

**適用方法**:
- シナリオ作成時: 顧客理念に照らして判断
- 施策決定時: 上司ではなく顧客理念・行動基準が決定
- 属人的判断排除: 「このメールはコストがかからないから全員に送れば良いのではないか」→顧客理念に反する

**前提条件/注意点**:
- 経営者も社員も同一に守る
- 一つの部門の都合で修正・追加・廃止不可
- 顧客も行動基準に沿ったアプローチになじみ、居心地良さを感じる

### 顧客育成モデル

**定義**:
顧客がどうすればファンになり、顧客として育っていくかの構造化されたモデル。

**構成要素**(自動車メーカー例):
1. **潜在顧客**: ブランド認知獲得
2. **見込顧客**: 資料請求、試乗、商談
3. **新規顧客**: 納車、使い方説明
4. **リピーター**: メンテナンス、部品購入
5. **ロイヤル顧客**: 買い替え、紹介

**GDOの顧客育成モデル**:
- 未登録カスタマー
- 未登録ノンカスタマー
- 登録 → 初回 → 二回目 → 三回目 → リピーター → ロイヤル顧客

**適用方法**:
- 最上位概念図からドリルダウンして詳細シナリオ作成
- フェーズ・プロセス・アクション・コンテンツで具体化
- 8つの視点でシナリオ分類

---

### 8つの視点シナリオ分類(GDO)

**テーマ①: 登録起点関係構築シナリオ**
- 方針: 我々を知ってもらう、ゴルフライフを豊かにする
- コンテンツ: 挨拶、自己紹介、サービス案内、ゴルフトリビア

**テーマ②: トランザクション起点関係構築シナリオ**
- 方針: 顧客との関係を作る、認知的不協和解消
- コンテンツ: 購入・予約お礼、商品・ゴルフ場情報、天気情報

**テーマ③: トランザクション起点商品ケアシナリオ**
- 方針: 認知的不協和解消
- コンテンツ: 商品メンテナンス情報、プレー後お礼・いたわり

**テーマ④: トランザクション起点注文獲得シナリオ**
- 方針: 顧客関心発見、注文誘導、欲求に合う商品提案
- コンテンツ: レコメンドメール、特別セール案内

**テーマ⑤: 定期注文獲得シナリオ**
- 方針: 同上
- コンテンツ: レコメンドメール、特別セール案内

**テーマ⑥: 定期サイト誘致シナリオ**
- 方針: 顧客をサイトに誘致
- コンテンツ: 定例セール案内、サイト更新情報、既存メルマガ

**テーマ⑦: 全体向け定期顧客ケアシナリオ**
- 方針: 顧客との関係維持
- コンテンツ: 季節挨拶、誕生日

**テーマ⑧: 特定客向け定期顧客ケアシナリオ**
- 方針: 特定客(優良客)との関係強化
- コンテンツ: 特定客向けイベント、カード、お中元

### ウェルカムシナリオ設計

**定義**:
会員登録直後から送り始める、顧客がGDOサービスを熟知し使えるようになるまでの導入シナリオ。

**構成**:
- 20数通のメールを1ヶ月かけて少しずつ配信
- 期間中は販促メール等、ウェルカムシナリオ以外のメールは配信しない

**反対意見と顧客理念による正当化**:
- 反対: 「一般的なウェルカムシナリオは3~5通、1週間以内」「やりすぎ」「長すぎる」
- 正当化: GDO規模のサイトはサービスが多い、顧客が熟知して使えるようになるまで1~2年放っておく方が問題。友人に勧めるなら「検索の使い方」「早特割」「別優待券」等、伝えるべき情報は3通では収まらない

### 確率論もどき思考の排除

**定義**:
「入口量を増やせば出口量が比例して増える」という誤った日本流思考。結果論の一種。

**問題点**:
1. **メール配信への適用**:
   - 総配信数 = 配信顧客数 × 平均配信回数
   - 顧客が毎日7~8通のメールを受け取る状態
   - 配信数のみ管理、顧客視点(読む気持ち)を無視

2. **営業への適用**:
   - 「6台売るには確率的に200軒訪問必要」(100軒で3台実績)
   - 200軒訪問すれば顧客接触時間は半分に減少
   - 確率3%は過去データ、未来の確率ではない(結果論)

3. **因果論もどきとの混同**:
   - V数(原因)100 → 受注数(結果)1 と考える
   - 実際はV数・CV数は「前アクションの結果」であって原因ではない
   - 受注の前アクションを改善することでしか受注数は改善しない

**GDO成功の背景**:
- 確率論もどき思考からの脱却
- 質で開封率を高め、読んでもらい、心を動かし、買ってもらう行動戦略樹立

### 普遍的プロセス構造

**定義**:
自然の摂理、法則に則って万物に共通する、変化に対応できるプロセス構造。

**4つの普遍的プロセス構造**:

**構造1: 階層構造**
- ① フェーズ(大分類、物語の章立て)
- ② ステップ(中分類、停留所名・マイルストーン)
- ③ アクション(詳細アクションの組み合わせ)
- ④ 詳細アクション(営業活動の具体的な1アクション、コンテンツ)

**構造2: 現在地の明確化**
- アクション、詳細アクションを現在形で捕捉する
- 顧客が今どのステップにいるかを特定

**構造3: 次の一手を考える**
- アクション、詳細アクションの進捗制御ができる

**構造4: 普遍性**
- 商品が変わり、競合が変わり、社会が変化しても変化しないプロセス構造
- BtoB・BtoC・デジタル・リアルを問わず適用可能

**適用方法**:
- RPG(ロールプレイングゲーム)と同一構造
- フラグ: プレイヤーが物語進捗ステップ何番目にいるか、どこまでイベント進捗しているかを把握
- アクション誘導: 「南の方に最近、魔物が出るようだ」(現在形)
- 現在地 → 詳細アクション → 次の目的地 → 最終目的達成

**RPGとの対比**:
| RPG | ビジネス |
|-----|---------|
| 物語の章立て(フェーズ) | 商談プロセスの大区分 |
| ステップ現在地 | 商談現在地 |
| イベントクリア | 詳細アクション完了 |
| フラグ | ログデータ(現在地+進捗状況) |
| 主人公 | 顧客 |
| ゲーム設計者 | 営業・マーケター |

### シナリオ設計(線路工事の比喩)

**定義**:
顧客とコミュニケーションの経路となるチャネル。情報を通すための線路。

**構成要素**(線路工事に例える):
1. **線路**: シナリオそのもの(情報を通す経路)
2. **車両**: 情報を載せる適切なメディア(メール、DM等)
3. **乗降ホーム**: 1シナリオでINとOUTの2ホーム必要
4. **発券機**: シナリオに乗れる権限のある顧客を制御
5. **改札口**: 乗客の乗降を制御
6. **信号機**: 複数の電車を正確に目的地へ到着させる
7. **駅員・運転手・乗客**: 登場人物(企業、マーケター、顧客)

**シナリオタイプ**:
1. **トランザクション起点**: 購買データ、ログデータをトリガー
2. **スケジュール起点**: 予定しているセール、イベント等

**シナリオ仕様設計図**(5W1Hベース):
- いつ、だれが、どこで、何を、どんな目的で、どうやって届けるか

### ログデータ活用と現在形分析

**定義**:
過去形データではなく、アクションと同時に取得したログデータを現在形で活用。

**過去形データとの違い**:
- **過去形**: 一定期間のデータを時系列に並べてグラフ化 → 結果論
- **現在形**: アクションの結果をリアルタイムで捕捉 → 次のアクション決定に使用

**活用方法**:
- 顧客の行動ログから現在地を自動特定
- 関係性レベルを計測
- 現在地に紐づいた詳細アクションを提示
- 次のプロセスへ進めるための鍵(フラグ)として機能

**前提条件/注意点**:
- Webマーケティング(不特定多数のマス・マーケティング)とは異なる
- デジタルONE to ONEマーケティングは顧客一人ひとりとの長期的関係構築が目的

## 成功事例

### GDOゴルフダイジェスト・オンライン: 3年でリード数30倍&V字回復

**背景**:
- 2011年東日本大震災で赤字転落
- 成熟化市場で新規顧客獲得困難
- RFM分析・データマイニング分析は失敗続き
- メール配信数増加 → 顧客離れの負の連鎖

**課題**:
- どんな会員が、いつ、どんな商品を購買するか分からない
- メルマガ全員配信というマス・アプローチ、販促精度低
- GDOを使い続ける理由を提供できていない
- 体系的コミュニケーションプランなし、場当たり的

**施策実行ステップ**:
1. **顧客理念・行動指針策定** (1ヶ月): 大勢の社員参加ディスカッション
2. **顧客育成モデル作成**: EC、予約、メディア、各サービス単位
3. **8つの視点シナリオ分類**: 70本のシナリオ設計
4. **原田尚子抜擢**: ゴルフオタク新卒2年目女子社員、文章に血が通っている
5. **フィジビリティスタディ**(2ヶ月): 新規登録1,000名対象(テスト500名+コントロール500名)
6. **経営会議承認**: 2条件付き(①現場理解獲得、②前年売上維持)
7. **システム選定**: 海外製ＭＡツール(ONE to ONEカスタマージャーニー実現)
8. **現場説得**: 分科会繰り返し、1対10の議論、2~3時間延長も
9. **リリース**: 段階的シナリオ導入、初回トラブル(同メール2通配信)

**結果(導入後半年)**:
| 指標 | 変化 |
|------|------|
| EC事業メール配信数 | -30% |
| EC事業CV数 | +35% |
| 予約事業メール配信数 | -50% |
| 予約事業CV数 | +45% |
| 新規顧客数 | +30% |

**フィジビリティスタディ結果**(2ヶ月間):
| 指標 | 向上率 |
|------|--------|
| ゴルフ場予約・EC購入双方稼働率 | +85.2% |
| メール開封率 | +75.1% |
| クリック率 | +34.0% |
| 1人あたり予約件数 | +9% |
| 予約後キャンセル率 | -14% |

**詳細分析**:
- テストグループ開封率: 初日比最終日+35%(コントロールは-5%)
- テストグループクリック率: 初日比最終日+92%(コントロールは-70%)
- 延べ配信数: -36%、延べ開封数: +13%
- 同一メール2回配信時の開封率: 1回目+80%、2回目+100%

**顧客の声(原田尚子宛て)**:
- 「原田さんのキャラクターや顔が見えるメルマガに親近感が沸きます」
- 「超初心者向け情報があると読んでみようかなと思います」
- 「結構面白いです。頑張ってください」
- 「パーソナルなコンシェルジュが付いたような感覚、非常に好感」
- 「プレー前日までの案内、とても助かりました」

**成功要因**:
1. **顧客理念の明確化**: 全アクションがブレない基準を持つ
2. **ビッグデータ活用**: 購買データ、予約データ、Web閲覧データ、メール反応データ、行動ログを組み合わせ、顧客状態・行動特性を読み解き、心地いいコミュニケーション実現
3. **コンテンツクオリティ**: 戦略・シナリオ・システムと同等に、最終顧客接点のコンテンツ・トークスクリプト・デザインに注力
4. **経営陣の強コミット**: 吉川取締役が「顧客の心に目を向けた新しいやり方で売上を伸ばす」と決断

## 日本流標準化への道

**個別化から標準化へ**:
- 欧米: 標準化文化(プラットフォーム、共通言語、OS)
- 日本: 個別化・カイゼン文化(島国で独自発展、俺流・私流、掘り下げ技術)

**日本的デジタルマーケティング成功パターン**:
1. 個別化から入る(俺流、私流で始める)
2. 勝ちパターンをログデータ解析で見つける
3. 優れた成功パターンに統一
4. ベターパターンを標準化(昨日よりベター、本日よりベターの繰り返し)
5. システム的にデジタルマーケティングに組み込む

**前提条件**:
- 普遍的なプロセス構造を持つ
- アクションを伴った時系列ログデータ
- ログデータを現在形で活用
- 営業プロセスの確立

## Webマーケティング vs デジタルONE to ONEマーケティング

| 観点 | Webマーケティング | デジタルONE to ONEマーケティング |
|------|-------------------|----------------------------------|
| 本質 | 不特定多数に対するマス・マーケティング | 顧客一人ひとりとの長期的関係構築 |
| 主役 | 過去形データ(From-To期間集計) | 現在形ログデータ(リアルタイム) |
| 目的 | 集客・最適化(SEO、LPO、EFO) | 顧客生涯価値(LTV)実現 |
| データ活用 | アクセス解析、効果測定、広告検証 | 顧客行動戦略、プロセス制御 |
| 手法 | バナー広告、リスティング広告 | シナリオ、オムニチャネル、関係性レベル計測 |
| 解析 | 結果論(過去データから仮説立案) | 制御(現在地と差分から次のアクション再設計) |
| 関係性 | Webサイト→顧客(集客強化) | 強力なパートナー(集客+関係深化) |

**相性の良さ**:
- Webマーケティング: 検索エンジンベースの集客が得意
- デジタルONE to ONEマーケティング: 集客した顧客と関係性を深め、維持・育成・離脱防止が得意
- 両者を組み合わせることで圧倒的な品質と効率向上
